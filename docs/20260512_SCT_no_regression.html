Run SCTransform with no covariate regressed

Published

May 24, 2026

We build a baseline single-cell clustering for the microglia_switch dataset. We apply SCTransform without regressing out any covariate, then run the standard graph-based clustering pipeline (PCA → KNN → Louvain).

SCTransform (no covariate regressed)

Code 
# eval:false because this block is the one-off build step; the checkpoint above is reloaded on every render.

# Load the raw merged object produced by R/20260512_xenium2seurat.R.
xen <- qs_read("data/20260512_microglia_switch_original_seurat.qs2")
# Xenium is the count assay; force it as default so SCTransform reads the right matrix.
DefaultAssay(xen) <- "Xenium"
cat("cells:",    ncol(xen),
    " features:", nrow(xen),
    " assays:",  paste(Assays(xen), collapse = ", "),
    "\n")

# Baseline normalization with no covariate regressed
xen <- SCTransform(xen, assay = "Xenium", verbose = FALSE)
# Switch default to SCT so downstream Run/Find* use the residuals, not raw counts.
DefaultAssay(xen) <- "SCT"
cat("variable features:", length(VariableFeatures(xen)), "\n")

# Default npcs (50) is fine; we only use the first 30 downstream.
xen <- RunPCA(xen)

# 1:30 PCs matches the QC plan; FindNeighbors default k = 20.
xen <- FindNeighbors(xen, reduction = "pca", dims = 1:30, verbose = FALSE)

# algorithm = 1 is plain Louvain (not Leiden); resolution 0.5 is the agreed coarse-grain setting.
xen <- FindClusters(xen, resolution = 0.5, algorithm = 1, verbose = FALSE)

# UMAP only for visual QC of the clusters; not used by any downstream stat.
xen <- RunUMAP(xen, reduction = "pca", dims = 1:30, verbose = FALSE)
